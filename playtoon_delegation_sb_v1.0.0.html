발행 권한 위임이란

워크스페이스 V1.2.1 IA의 ⑤ 팀·멤버 관리 > 발행 권한 위임 서브영역

팀장이 부재하거나 직접 검수 제출이 어려울 때, 팀원에게 발행 관련 액션 권한을 한시적·범위 제한적으로 위임하는 영역. 작품의 author는 팀장 고정으로 유지되고, 위임받은 팀원은 "토큰"의 형태로 위임 권한을 수령한다.

신규 영역 (V1.2.1) 4개 서브항목 Author 불변 원칙 CP 마스터 케이스 포함
A1 위임 모델의 6대 원칙 모든 화면과 흐름의 기반 정책

본 SB의 모든 화면은 다음 6개 원칙을 기반으로 설계되었다. 한 원칙이 어떻게 UI로 표현되는지 각 챕터에서 다룬다.

P-1
토큰형 권한
위임은 "권한 부여"가 아닌 시한·범위가 있는 토큰. 회수·만료 가능. 작품을 "양도"하지 않는다.
P-2
발급자 제한
팀장(team.owner_author_id) 본인. CP 소속 팀의 경우 CP 마스터(AUTHOR.role=master)도 발급 가능 (E1 참조).
P-3
수령자 제한
같은 팀의 active 멤버에게만 위임 가능. 팀 외부 작가나 익명에게는 위임 불가. 멤버 권한(Owner/Editor/Commenter/Viewer)과 무관하게 수령 가능.
P-4
Author 불변
위임받은 팀원이 업로드해도 작품의 author는 팀장 고정. 외부(독자) 표기는 변하지 않는다. uploader 정보는 어드민·팀 내부에서만 추적 가능.
P-5
이력 추적
발급·연장·회수·만료·사용의 모든 이벤트가 AUTH_AUDIT_LOG에 기록. 어드민에서 전체 이력 조회 가능. 작가는 본인 발급 이력만 (F 참조).
P-6
액션 범위 명시
일괄 권한 부여가 아님. submit_review · schedule_publish · edit_publish_history 등 액션 타입을 명시해서 발급. default는 submit_review만 (가장 빈번한 케이스).

발급 폼 흐름 개요

위임 발급은 5개의 단계로 구성된 모달 시퀀스다. 발급자가 선택해야 할 4가지 차원(수령자·범위·액션·기간)을 한 단계씩 결정한 뒤 최종 확인하고 발급한다. 잘못된 조합은 자동 검증·차단된다.

일관된 시각 언어 — Lavender

수령자 측의 모든 위임 관련 UI는 Lavender 톤으로 일관성 유지. 발급자 측은 Brand Red. 사용자는 컬러만 봐도 "내가 위임받은 것"과 "내가 발급한 것"을 구분 가능.

정책 변경 시 동작

  • CP 마스터가 토글 변경 → 확인 다이얼로그 + 이유 입력 (audit log용)
  • 소속 모든 작가에게 이메일 + 인앱 알림 발송
  • 알림 본문: "CP-001의 무위임 업로드 정책이 ON으로 변경되었습니다. 동의하지 않는 경우 CP 탈퇴를 요청할 수 있습니다."
  • Audit Log에 cp_policy_changed 이벤트 기록 (이전 값, 새 값, 사유 포함)

왜 OFF가 권장인가

작가의 작품에 대한 통제권을 보장하는 것이 기본 원칙. ON 모드는 회사 정책에 따라 작가가 모르는 사이에 업로드가 일어날 수 있어, 분쟁 시 작가 보호가 약해진다. 부득이한 경우 (예: 일괄 마이그레이션·운영 자동화)에만 임시 ON 권장.
