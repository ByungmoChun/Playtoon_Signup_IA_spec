왜 Primary가 Lavender인가

  • 워크스페이스 디자인 시스템상 라벤더는 "선택적 액션·다음 단계" 컬러
  • 빨강은 "신규 작성·실행" 컬러 (CTA의 기본)
  • 연동은 선택 가능한 다음 단계이므로 라벤더 채택 (실제 설정 페이지의 [저장] 버튼과 동일 컬러)

유의사항

  • 두 계정의 비밀번호는 동일 입력값으로 시작하지만 이후 독립 관리 (Q2)
  • 닉네임은 작가 필명과 분리하는 것을 권장 (Q7 정체성 분리)
  • 사후 연동은 워크스페이스 [설정 > 연결된 계정]에서 가능

해제 4단계

Step처리 내용
1. 상태 확인현재 연동된 상대편 계정 정보 표시 (이메일 마스킹) + 연동일
2. 확인 다이얼로그해제 결과 안내 + 비밀번호 재입력 (사회공학 공격 방지)
3. 해제 처리LINKED_ACCOUNT.status = unlinked, unlinked_at = NOW(), unlinked_reason 저장
4. 완료 알림양쪽 계정 이메일 알림 · "본인이 한 해제가 아니라면 비밀번호 변경 권고"

해제 후 상태

  • USER · AUTHOR 데이터 모두 그대로 유지 (구매 이력 · 작품 · 잔액 등)
  • LINKED_ACCOUNT 레코드는 status=unlinked로 남아 이력 보존
  • 로그인은 양쪽 모두 그대로 작동 (각자 독립 비밀번호이므로 Q2)
  • 다시 연동하려면 C2 사후 연동 흐름 재실행

UX 측면 유의사항

  • 해제는 신중한 액션 — Primary가 아닌 Danger 색상 (Error Red)
  • 데이터가 유지됨을 명확히 안내 (사용자 안심)
  • 운영자 측에서 모든 해제 이력이 audit log로 추적됨
  • 같은 계정 쌍의 잦은 연동/해제 반복은 시스템 알림 발동

왜 워크스페이스만 Naver 없나

  • 작가 페르소나는 한국 외 시장도 잠재 고객 (글로벌 작가 진입 가능)
  • Google·Apple은 글로벌 표준, Kakao는 한국 작가 편의 — 충분한 커버리지
  • Naver는 한국 독자 채널로 더 적합하다고 판단
  • 실제 dev 환경의 로그인 모달에서 Naver가 빠진 것 확인

중요 — 독립 비밀번호 정책 (Q2)

USER DB와 AUTHOR DB는 비밀번호가 독립적으로 관리된다. 같은 이메일로 양쪽 가입한 사용자라도 양쪽 비밀번호가 다를 수 있으며, 재설정은 진입 페이지(워크스페이스/플랫폼)에 따라 분기된다.

분기 처리

진입대상 DB재설정 메일 제목
workspace.playtoon.io/resetAUTHOR[Playtoon] 작가 계정 비밀번호 재설정
playtoon.com/resetUSER[Playtoon] 독자 계정 비밀번호 재설정

처리 단계

Step처리
1. 이메일 입력가입된 이메일 입력 · 양쪽 DB에 없어도 일관된 응답 (사용자 존재 노출 방지)
2. 토큰 발급PASSWORD_RESET_TOKEN (target_db: user 또는 author) · 1시간 만료
3. 메일 발송재설정 링크 + DB 식별 명시
4. 사용자 재설정새 비밀번호 입력 · 강도 검증 · 해당 DB의 password_hash 갱신
5. 완료 알림변경 완료 메일 발송 + 모든 세션 무효화

유의사항

  • 이메일이 양쪽 DB에 모두 존재하는 경우, 진입한 페이지의 DB만 재설정 (다른 쪽은 영향 없음)
  • 토큰은 1회만 사용 가능 · 발급 후 1시간 만료
  • 재설정 후 모든 활성 세션 강제 종료 + 알림
  • 재설정 메일 발송은 5분 쿨다운 · 일 5회 제한

"어느 계정인지 모르겠어요" 케이스

사용자가 양쪽 계정 중 어느 쪽 비밀번호를 잊었는지 불명
→ 양쪽 페이지(workspace · platform)의 [비밀번호 찾기] 진입점에 안내문 노출 → 사용자가 현재 페이지의 종류를 인지하고 진행

다크 모드 운영 원칙

  • · Brand Red는 다크 모드에서도 동일하게 유지 (브랜드 일관성)
  • · Maroon 카드 배경은 다크 모드에서 더 어둡게 (#5E1717 → #4A1515)
  • · Lavender는 다크에서도 동일 (Q3 정책 일관성)
  • · 회원가입 화면은 라이트 모드 default 권장 (가입 시점은 진입자의 첫 접점이므로 밝은 톤이 신뢰감)
  • · 단, 사용자 OS 설정이 다크 모드이고 명시적 라이트 선택이 없다면 다크 모드로 진입 가능 (시스템 설정 존중)
  • · 로그인 모달의 좌측 일러스트 영역은 다크 모드에서도 동일한 컬러 노출 (콘텐츠 자체가 색감을 가지므로 영향 없음)
